# worker-brief — worker サブエージェント委任ブリーフ

agmsg 経由で worker サブエージェントに実装を委任するときは、以下のブリーフテンプレートを埋めてから送る。worker の「勝手なアイコン追加」「指示外の実装修正」「停止指示・メッセージの無視」は委任における最大の摩擦源であり、自由記述の一文では防げない。ブリーフの厳格さで防ぐ。

## 手順

1. **タスクを 1 worker = 1 スコープに切る**。曖昧な「いい感じにして」を worker に投げない。何を変え、何を変えないかを列挙する。
2. 下のテンプレートを埋める。特に「やらないこと」「中断ルール」「セルフチェック」は削らない。
3. 委任後、worker からの完了報告はそのまま信用せず、セルフチェック項目が満たされているかを確認する。UI 実装の場合は、デザイン基準との視覚検証を worker にも課す。

## ブリーフテンプレート

```
## タスク
<何を実装するか。対象ファイル・画面・エンドポイントを具体的に>

## スコープ（これだけをやる）
- <変更してよいファイル/範囲を列挙>

## やらないこと（厳守）
- 依頼範囲外のファイルを変更しない。
- 勝手な装飾・アイコン・ラベル・文言を追加しない（例: タイトル横にアイコンを足す等は禁止）。
- リファクタリング・「ついでの改善」を勝手にやらない。必要と感じたら実装せず報告する。
- 対象ファイルは自分で Read してから編集する（親が読んでいても子は再 Read が必要）。

## 中断・メッセージ受信ルール（最優先）
- boot 直後に `~/.agents/skills/agmsg/scripts/delivery.sh status claude-code "$(pwd)"` で delivery mode が `monitor`（または `both`）であることを確認し、SessionStart directive で起動した agmsg inbox Monitor を**セッション終了まで維持**する。mode が `off`/`turn` と表示されたら（orchestrator 側の設定漏れ）、**作業を始める前に orchestrator へ報告**する。
- 作業中にメッセージ（agmsg/tmux）を受信したら、**その時点で作業を中断し、内容を確認してから**続行する。無視して作業を続けない。
- 「止まって」「stop」「待って」等の停止指示には即座に応じ、現在の状態を報告する。
- 完了報告を送った後も Monitor を止めない。追加依頼（レビュー対応等）が届く前提で待機する。

## 完了報告前のセルフチェック（全項目 OK になるまで報告しない）
- [ ] agmsg inbox Monitor（`agmsg inbox stream`）が稼働中である
- [ ] スコープ内のファイルだけを変更した（範囲外の diff が無い）
- [ ] 勝手な追加要素（アイコン・装飾・文言）が無い
- [ ] （UI の場合）指定された視覚検証を通過した
- [ ] 受信したメッセージ・停止指示に全て応答済み
- [ ] 変更点を簡潔に列挙できる（何を変えたか説明できる）

## 報告フォーマット
- 変更ファイル一覧 / 変更内容の要約 / セルフチェック結果 / 未対応・懸念点
```
